================================================================================
TROUBLESHOOTING
================================================================================

If you're still getting 404 errors:

1. Check Build Logs
   [ ] Go to Deployments → Latest → View Build Logs
   [ ] Look for "Build failed" or error messages
   [ ] Fix any build errors and redeploy

2. Verify Environment Variables
   [ ] Go to Settings → Environment Variables
   [ ] Confirm all 6 variables are present
   [ ] Confirm they're set for "Production" environment
   [ ] Check that values are correct (no typos)

3. Test API Endpoints
   [ ] Try /api/test first (doesn't need database)
   [ ] If /api/test returns 404, API routes aren't being served
   [ ] If /api/test works, issue is with page rendering

4. Check Database Connection
   [ ] Try /api/health endpoint
   [ ] If it shows "database": "disconnected", DATABASE_URL is wrong
   [ ] Verify DATABASE_URL in Supabase dashboard

5. Apply Database Migrations
   [ ] If health check shows unhealthy, migrations may not be applied
   [ ] Run: npx prisma migrate deploy
   [ ] Or manually run SQL from prisma/migrations/001_init.sql

⚡ OPTION 1: VERCEL + SUPABASE (FASTEST - 5 MINUTES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Best for: Quick deployment, minimal setup, free tier available

STEP 1: Create Supabase Database (2 min)
   1. Go to https://supabase.com
   2. Click "New Project"
   3. Fill in project details
   4. Wait for creation (2-3 minutes)
   5. Copy connection string
   6. Save as DATABASE_URL

STEP 2: Create Upstash Redis (1 min)
   1. Go to https://upstash.com
   2. Click "Create Database"
   3. Choose "Redis"
   4. Copy Redis URL
   5. Save as REDIS_URL

STEP 3: Deploy to Vercel (2 min)
   1. npm install -g vercel
   2. vercel login
   3. vercel --prod
   4. Follow prompts

STEP 4: Set Environment Variables (2 min)
   1. Go to Vercel Dashboard
   2. Select your project
   3. Settings → Environment Variables
   4. Add these:
      - DATABASE_URL (from Supabase)
      - REDIS_URL (from Upstash)
      - JWT_SECRET (generate: openssl rand -base64 32)
      - JWT_REFRESH_SECRET (generate: openssl rand -base64 32)
      - ENCRYPTION_KEY (generate: openssl rand -hex 16)
      - OPENAI_API_KEY (optional)
      - NEXT_PUBLIC_API_URL (your Vercel domain)
      - NODE_ENV = production

STEP 5: Run Migrations (2 min)
   npx prisma migrate deploy --skip-generate
   npx prisma db seed

STEP 6: Verify (1 min)
   curl https://YOUR_VERCEL_DOMAIN.vercel.app/api/health

STEP 7: Access Your App (1 min)
   1. Open: https://YOUR_VERCEL_DOMAIN.vercel.app
   2. Login: admin / admin123
   3. Done! ✅

TOTAL TIME: 5 minutes

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🐳 OPTION 2: DOCKER SELF-HOSTED (10 MINUTES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Best for: Full control, custom domain, self-hosted

PREREQUISITES:
   - VPS with Docker (DigitalOcean, Linode, AWS EC2, etc.)
   - SSH access to server
   - Domain name (optional)

STEP 1: SSH into Server (1 min)
   ssh root@YOUR_SERVER_IP

STEP 2: Clone Repository (1 min)
   cd /opt
   git clone YOUR_REPO_URL smart-pos-system
   cd smart-pos-system

STEP 3: Create Production Environment (2 min)
   cp .env.production.example .env.production
   nano .env.production
   
   Set these values:
   - DATABASE_URL=postgresql://postgres:PASSWORD@localhost:5432/smart_pos
   - REDIS_URL=redis://localhost:6379
   - JWT_SECRET (generate: openssl rand -base64 32)
   - JWT_REFRESH_SECRET (generate: openssl rand -base64 32)
   - ENCRYPTION_KEY (generate: openssl rand -hex 16)
   - OPENAI_API_KEY (optional)
   - NEXT_PUBLIC_API_URL=https://your-domain.com

STEP 4: Start Services (2 min)
   docker-compose -f docker-compose.production.yml up -d
   docker-compose -f docker-compose.production.yml ps

STEP 5: Run Migrations (2 min)
   docker-compose -f docker-compose.production.yml exec app npx prisma migrate deploy
   docker-compose -f docker-compose.production.yml exec app npx prisma db seed

STEP 6: Setup SSL (Optional, 2 min)
   apt-get update
   apt-get install certbot python3-certbot-nginx
   certbot --nginx -d your-domain.com

STEP 7: Verify (1 min)
   curl http://localhost:3000/api/health

STEP 8: Access Your App (1 min)
   1. Open: http://YOUR_SERVER_IP or https://your-domain.com
   2. Login: admin / admin123
   3. Done! ✅

TOTAL TIME: 10 minutes

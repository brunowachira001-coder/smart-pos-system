⚡ QUICK STEPS (2 MINUTES):

  1. Log in to Supabase
  2. Select your project
  3. Click "Settings" (bottom left)
  4. Click "Database"
  5. Click "Connection Pooling" tab
  6. Click "Copy" button
  7. Done!

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 LOCATION IN SUPABASE:

  Settings → Database → Connection string

📋 CONNECTION STRING FORMAT:

  postgresql://postgres:PASSWORD@db.PROJECT_ID.supabase.co:5432/postgres

  Example:
  postgresql://postgres:Tr0pic@lSunset#2024!Secure@db.abcdefghijklmnop.supabase.co:5432/postgres

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔑 CONNECTION STRING COMPONENTS:

  postgresql://     ← Protocol (PostgreSQL)
  postgres          ← Username
  :PASSWORD         ← Your database password
  @db.xxx.supabase.co  ← Supabase host
  :5432             ← Port number
  /postgres         ← Database name

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ TWO CONNECTION MODES:

  1. Connection Pooling (Recommended for Vercel)
     ✓ Better for serverless
     ✓ Handles connection limits
     ✓ More reliable
     → USE THIS ONE!

  2. Direct Connection (For local development)
     ✓ Better for persistent connections
     ✓ Use for development only

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💾 WHERE TO SAVE CONNECTION STRING:

  Option 1: .env.local file
  ┌─────────────────────────────────────┐
  │ DATABASE_URL=postgresql://postgres: │
  │ PASSWORD@db.xxx.supabase.co:5432/   │
  │ postgres                            │
  └─────────────────────────────────────┘

  Option 2: Vercel Environment Variables
  ┌─────────────────────────────────────┐
  │ Name: DATABASE_URL                  │
  │ Value: postgresql://postgres:...    │
  └─────────────────────────────────────┘

  Option 3: Password Manager
  ┌─────────────────────────────────────┐
  │ 1Password / LastPass / Bitwarden     │
  │ Save connection string securely      │
  └─────────────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔐 SECURITY TIPS:

  ✓ Never commit to Git
  ✓ Don't share with others
  ✓ Use environment variables
  ✓ Save in password manager
  ✓ Rotate password regularly
  ✓ Don't hardcode in code

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ SPECIAL CHARACTERS IN PASSWORD:

  If password has special characters, URL encode them:

  @ → %40
  # → %23
  ! → %21
  : → %3A
  $ → %24
  & → %26

  Example:
  Password: Tr0pic@lSunset#2024!Secure
  Encoded: Tr0pic%40lSunset%232024%21Secure

❓ TROUBLESHOOTING:

  Can't find Connection string?
  → Go to Settings → Database
  → Look for "Connection string" section
  → Click "Connection Pooling" tab

  Connection string looks wrong?
  → Should start with: postgresql://postgres:
  → Should end with: :5432/postgres
  → Refresh page and try again

  Password not showing?
  → It's hidden for security
  → Use the password you set when creating project
  → If forgotten, reset in Database settings
